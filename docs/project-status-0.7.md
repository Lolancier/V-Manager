# V-Manager 0.7 阶段状态与后续规划

> 状态日期：2026-07-22
>
> 主线状态：阶段完成，暂时停止新增功能
>
> 当前版本：0.7.0

## 项目定位

V-Manager 当前是一套 Windows 桌面二次元陪伴 Agent。它以 Live2D 角色为交互载体，组合了自然对话、本地记忆、关系成长、语音、表情动作、桌面工具和角色化代码工作台。

本阶段的目标不是继续扩大功能数量，而是保存一套已经能够构建、测试和继续演进的主线基准。

## 已完成能力

### 陪伴与对话

- DeepSeek 兼容接口的日常对话与复杂任务模型分流
- 本地多轮记忆、知识库检索、记忆压缩和关系成长
- 聊天工作台、桌面气泡、快捷输入和角色触碰互动
- 本地 Whisper 语音输入、ElevenLabs 语音输出和 ASMR 文本工作区

### Live2D

- 内置模型、用户模型扫描、运行时热切换和模型能力探测
- LLM 情绪与表情参数驱动、口型同步和手动表情面板
- 双阶段反应：收到消息时先做本地预反应，完整回复后再表演最终情绪
- 参数混合层协调情绪、动作、口型、鼠标注视和手动覆盖
- 鼠标注视不再依赖指针位于模型本体范围，并提供菜单快捷开关

### 桌面与代码能力

- 系统资源、应用启动/关闭、文件操作、浏览器搜索、VS Code 和实验性微信代发
- 32 个 Function Calling 工具，按意图限制可见工具集合
- Vivi Code 五模式：自动、问答、规划、Agent、审查
- 工作区搜索、读取、精确替换、整文件安全更新、新建文件和受限开发命令
- 直接代码编辑、Ctrl+S 保存、并发覆盖保护、工作区边界与权限控制
- 代码会话继承 Vivi 的人设与关系上下文，而不是独立的无人格执行器

## 本阶段验证

- `npm run build`：通过
- `npm test`：24 项测试全部通过
- `git diff --check`：通过（仅存在 Windows 换行提示）
- Electron 构建产物已迁移到项目目录外，避免旧 `app.asar` 被索引程序锁定

## 已知限制

- Vivi Code 仍是单进程对话任务，不具备完整后台 Job 队列、暂停、取消、重试和可视化 Trace
- 代码命令采用安全白名单，不等同于 Claude Code 的完整终端权限
- 通用 Live2D 参数映射依赖模型规范；非标准模型可能需要用户提供参数别名或专用配置
- 微信/AstrBot 仍属于实验性接入，读取回复、连续对话和 QQ 适配尚未完成
- 尚未建立完整 Electron 端到端自动化测试、安装包签名与自动发布流水线

## 后续开发规划

以下内容仅作为下一阶段候选，不代表当前承诺或排期。

1. 后台 Agent 任务系统：Job 状态、步骤计划、暂停、取消、重试、Trace 和阶段结果通知。
2. 通用 Live2D 模型配置：参数自动识别、映射编辑器、模型配置导入导出和兼容性报告。
3. 多设备接入：profile 初始化、配置迁移、知识库同步和隐私边界设计。
4. 消息联动：AstrBot 稳定路由、微信回复读取、连续对话和 QQ 适配。
5. 代码工作台深化：变更 Diff 视图、文件新建/重命名、任务检查点和 Git 提交流程。
6. 工程化：Electron E2E、崩溃恢复、性能监控、安装包签名和自动发布。

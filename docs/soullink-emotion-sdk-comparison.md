# SoulLink Emotion SDK 实时表情对比

参考仓库：`references/soullink-emotion-sdk`（MIT License）。

## 结论

V-Manager 不适合直接替换成 SoulLink 的整套运行时。当前项目使用 Live2D 官方 Cubism Web SDK，参考项目的渲染适配层使用 PixiJS / `pixi-live2d-display`，直接套包会造成渲染栈重复、Pixi 主版本不一致，并绕开现有的官方 SDK 参数句柄、动作调度和模型加载逻辑。

适合复用的是它的设计方法：先按优先级合并情绪、反应、语音和手动层，再对最终参数做与帧率无关的指数平滑，并在效果结束后渐进回到中性值。

## 已接入

- 保留现有优先级：情绪预设 → 情绪组合 → 待机道具 → 手动表情 → LLM face 参数 → 口型。
- 在最终参数写入 Cubism 模型前加入统一的按帧平滑。
- 不同参数采用不同响应速度：口型最快，眼球和装饰参数居中，头部角度更柔和。
- 表情取消后先平滑回到中性值，再释放 override，让模型原生眨眼、动作和物理层重新接管。
- 切换模型和卸载组件时清空混合器状态，避免上一模型的参数残留。

## 后续可继续借鉴

- VAD 三维情绪（愉悦度、唤醒度、支配度）及短时保持/衰减，可替代当前较离散的 mood 跳转。
- 模型能力检测和参数 profile，可让第三方模型自动适配，而不是只依赖芊芊的参数白名单。
- 音频 RMS/peak 的 attack/release 口型，可替代当前纯正弦模拟口型。

这些部分需要分别接入关系状态、模型扫描和真实 TTS 音频节点，适合后续分阶段实现，不应直接复制整套运行时。

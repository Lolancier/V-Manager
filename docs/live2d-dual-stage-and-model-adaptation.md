# Live2D 双阶段反应与多模型适配

## 双阶段反应

每条用户消息现在会经过两阶段：

1. Electron 主进程使用本地高频语义规则生成低强度预反应，不等待网络和 LLM。
2. LLM 回复完成后，下发正式 mood、face 参数和回复表演，前端通过参数混合器平滑校正。

预反应覆盖亲近、惊讶、生气、低落/压力、积极表达、疑问六类；没有命中时使用轻微注视和歪头。预反应只持续约 0.7～1.1 秒，随后进入思考状态，不会提前启动口型。

## 多模型适配

用户仍可把模型目录放入应用数据目录下的 `agent-data/models`，每个模型至少需要完整的 `.model3.json`、`.moc3` 和纹理文件。

扫描器会展示以下能力：

- 原生 expression 数量；
- motion 分组数量；
- 是否声明 LipSync；
- 是否声明 EyeBlink；
- 是否提供 CDI DisplayInfo。

模型加载后，渲染器从 moc 中读取真实参数 ID，并自动完成：

- Cubism 3/4 标准 ID 与 Cubism 2 大写 ID 的映射；
- 只向模型写入真实存在的参数；
- 左右眉参数同步；
- 通用开心、难过、惊讶、生气、害羞、思考和说话姿态；
- 根据名称匹配 `Normal`、`Smile`、`Sad`、`Angry`、`Surprised`、`Blushing` 等原生 expression；
- 缺少某项能力时安全降级，而不是生成不存在的参数。

芊芊继续使用专属表情编号和组合；其他模型优先使用通用参数与自身原生 expression。

## 后续扩展点

- 为无法从名称判断的 `f01`、`exp_01` 等 expression 提供模型配置界面。
- 从 CDI 参数中文名和参数范围生成可编辑 profile。
- 加入 embedding 快速分类器，处理反讽和措辞变化更大的预反应。
- 将真实 TTS RMS/peak 接入口型 attack/release 控制器。

## 系统级鼠标注视

鼠标注视与点击/拖拽已分离。Electron 主进程约每 33ms 读取一次系统光标，并分别换算成桌宠窗口或聊天窗口的内容坐标。渲染端以 canvas 中心为原点生成注视方向；光标超出 canvas、模型轮廓或透明区域时，方向会归一化到单位圆，而不是停止更新或继续放大。

因此跟随在以下情况仍然有效：

- 鼠标离开 Live2D 模型本体；
- 鼠标位于桌宠窗口的透明穿透区域；
- 鼠标位于桌宠窗口外；
- 鼠标移动到另一块显示器。

设置页的“鼠标注视跟随”默认开启。关闭时会立即把注视目标平滑交还给模型中心和原生 Idle 动作。
